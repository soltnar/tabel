# История версий

## 1.4.7 (2026-04-02)
- Упрощен сценарий загрузки: файл `Табель` больше не обязателен, основной источник — расчетные листки.
- В `POST /upload` оставлен обязательным только `payroll_file` (плюс опциональный `employees_file`).
- Экспорт Т-13 переведен на встроенный формат (без шаблонного файла табеля).
- Календарь для генерации при отсутствии табеля берется как полный месяц `1..31` с базовой эвристикой выходных.

## 1.4.6 (2026-04-02)
- Усилена равномерность распределения дней по месяцу:
  - вместо выбора «первых N дней» применяется равномерный отбор по всей длине месяца.
- Для core-групп (`Кухня`, `Зал/Касса`, `Бар`) добавлен приоритет выходных при отборе дней в Т-13 и предпросмотре.
- Логика покрытия групп по ресторанам закреплена:
  - `Кухня`, `Зал/Касса`, `Бар` всегда учитываются в каждом ресторане;
  - если в группе нет доступных сотрудников, это отображается как дефицит.
- В предпросмотре скрыта колонка `Время`.

## 1.4.5 (2026-04-02)
- Исправлена логика покрытия групп по ресторанам:
  - `Кухня`, `Зал/Касса`, `Бар` теперь считаются обязательными группами для каждого ресторана.
  - если в ресторане нет сотрудников группы, смена фиксируется как дефицит.
- В таблице предпросмотра:
  - убрана колонка `Время`;
  - статус теперь берется из данных строки (`План из расчетного листка`, `ОК`, `Межресторанная замена`, `ДЕФИЦИТ`), а не подставляется как `ОК` по умолчанию.

## 1.4.4 (2026-04-02)
- Добавлена пагинация предпросмотра назначений в веб-интерфейсе:
  - первая порция после генерации;
  - кнопка `Загрузить еще` для подгрузки всех смен.
- Добавлен API endpoint `GET /preview?offset=&limit=` для постраничной выдачи всех записей графика.
- В `POST /generate` добавлены поля:
  - `preview_total`,
  - `preview_page_size`,
  - `preview_next_offset`.

## 1.4.3 (2026-04-02)
- Добавлен UI-редактор групп взаимозаменяемости должностей:
  - `Кухня`, `Зал/Касса`, `Бар`, `Обслуживание`.
- Добавлена передача пользовательской карты групп в `POST /generate`.
- Генератор теперь строит пулы взаимозаменяемости по выбранным в UI группам.
- Добавлена валидация допустимых групп на бэкенде.
- Обновлена базовая авто-классификация ролей:
  - админ/касса и зал -> `Зал/Касса`;
  - кухня -> `Кухня`;
  - бар -> `Бар`;
  - прочие -> `Обслуживание`.

## 1.4.13 (2026-04-02)
- Исправлено заполнение Т-13 в шаблоне:
  - добавлена строгая проверка валидности 4-строчного блока сотрудника по полной сетке дней;
  - исключено попадание записи в “кривые”/служебные зоны бланка, что убирает белые поля и сдвиги верстки.
- Дополнительно закреплена логика универсального фронт-персонала:
  - `менеджер` и `администратор` могут одновременно закрывать `Касса + Бар` в одну и ту же смену без двойного списания часов.

## 1.4.12 (2026-04-02)
- Обновлена логика обязательного покрытия core-групп (`Кухня`, `Зал`, `Касса`, `Бар`):
  - теперь используется один обязательный дневной слот на группу/ресторан/день;
  - дублирование обязательного покрытия по нескольким сменам отключено.
- Результат: меньше избыточных назначений в preview при сохранении контроля по дням/часам из расчетных листков в Т-13.

## 1.4.11 (2026-04-02)
- Шаблон Т-13 перенесен в проект: `templates/t13_template.xlsx`.
- Экспорт Т-13 теперь использует локальный шаблон из проекта по умолчанию (без зависимости от пути в `Downloads`), что корректно для серверного деплоя.
- Обновлена документация по источнику шаблона Т-13.

## 1.4.10 (2026-04-02)
- Улучшен экспорт Т-13 для максимальной унификации формы:
  - добавлен автоподхват шаблона Т-13 из файла `.xlsx` (приоритетно используется при выгрузке);
  - при наличии шаблона Т-13 заполняется непосредственно в бланке, с сохранением структуры формы.
- В резервном режиме экспорта убран формат `Я@R..`:
  - в ячейках дней теперь используется классический код `Я`;
  - сведения о межресторанных заменах остаются в примечаниях/комментариях.
- В статус/предупреждения генерации добавлено сообщение о том, какой шаблон был использован для Т-13.

## 1.4.9 (2026-04-02)
- Обновлена модель групп взаимозаменяемости:
  - `Зал/Касса` разделена на две отдельные группы: `Зал` и `Касса`.
- Добавлены новые правила подмен:
  - для `Бара` разрешены межресторанные подмены (аналогично кухне);
  - для `Бара` разрешена подмена сотрудниками группы `Касса` (в первую очередь локально, затем межресторанно).
- Обновлены дефолтные назначения ролей:
  - `администратор`/`кассир`/`менеджер` -> `Касса`;
  - `официант`/`хостес`/`раннер` -> `Зал`.
- Добавлена совместимость со старым значением `Зал/Касса` в API: автоматически маппится в `Касса`.

## 1.4.8 (2026-04-02)
- Добавлен учет официальных переносов выходных и праздничных дней РФ при генерации:
  - календарь месяца определяется автоматически из расчетного листа;
  - выходные считаются как `сб/вс` + праздничные/перенесенные нерабочие дни.
- Источник праздничного календаря: библиотека `holidays` (`RU`, `observed=True`) с fallback на фиксированные федеральные даты.
- Добавлены/зафиксированы дефолтные группы ролей:
  - `бар-менеджер` -> `Бар`;
  - `директор` -> `Зал/Касса`;
  - `управляющий по производству и учету` -> `Кухня`;
  - `подсобный рабочий` -> `Кухня`.

## 1.4.2 (2026-04-02)
- Исправлен экспорт Т-13: целевые `дни` и `часы` теперь берутся строго из расчетного листка (`max_days` / `max_hours`).
- Добавлено распределение часов по дням с поддержкой неровных значений (не только фиксированные 4/8).
- Экспорт Т-13 переведен на шаблонный режим:
  - в качестве формы используется загруженный файл табеля;
  - структура бланка (4 строки на сотрудника, шапка и сетка) сохраняется как в исходной форме;
  - фактические смены помечаются `Я@Rxx`, дни только из расчетного листка без назначенной смены — `ПЛ`.

## 1.4.1 (2026-04-01)
- Добавлен `render.yaml` для деплоя в Render Blueprint в 1 клик.
- Конфигурация запуска для облака зафиксирована как:
  - `uvicorn app.main:app --host 0.0.0.0 --port $PORT`.

## 1.4.0 (2026-04-01)
- Добавлены новые правила генерации:
  - `бармен` + `бар-менеджер` (взаимозаменяемость);
  - `официант` + `хостес` + `раннер` (взаимозаменяемость);
  - необязательные роли с распределением по будням/выходным.
- Смены обновлены под операционный график:
  - бизнес-ланч `12:00-16:00`;
  - вечерняя посадка `18:00-22:00`.
- В выходные по обязательным ролям усиливается покрытие.
- Добавлен экспорт табеля Т-13 с деталями по сменам и площадкам в ячейке дня.
- Добавлено версионирование через файл `VERSION` и endpoint `/version`.
- Добавлено файловое логирование:
  - общий лог `logs/app.log`;
  - отдельные логи ошибок `logs/errors/ERR_*.log`;
  - endpoints `/download_log`, `/download_error_log/{error_id}`.

## 1.3.0 (2026-04-01)
- Добавлен отдельный экспорт `Т-13`.
- Добавлены фильтры preview по дефициту и межресторанным подменам.
