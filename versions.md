# История версий

## 1.4.22 (2026-04-04)
- Добавлена генерация **заполненного PDF Т-13** на основе тех же данных, что используются для Excel Т-13:
  - сотрудники, дни, явки/часы и итоги формируются из `_build_t13_dataframe`.
- Добавлен endpoint `GET /download_t13_pdf` для скачивания заполненного PDF.
- В UI добавлена кнопка `Скачать табель Т-13 (PDF)` (активируется после генерации).
- Сохранена отдельная кнопка для пустого PDF-бланка: `Скачать форму Т-13 (PDF)`.
- В `requirements.txt` добавлен `reportlab==4.4.4`.

## 1.4.21 (2026-04-04)
- Добавлена отдельная PDF-форма Т-13 в проект: `templates/t13_form.pdf`.
- Добавлен endpoint `GET /download_t13_pdf_form` для скачивания PDF-формы напрямую из интерфейса.
- В UI добавлена кнопка `Скачать форму Т-13 (PDF)` в блоке скачивания результата.
- Версия обновлена до `1.4.21`.

## 1.4.20 (2026-04-04)
- Доработано контекстное «вырезание» на листе `Т-13 общий`:
  - служебные межстраничные блоки теперь определяются строго по контексту соседних строк сотрудников (`№`, `ФИО(роль)`, `табельный номер`);
  - между соседними сотрудниками скрываются только лишние служебные строки, сама табличная часть сотрудников не затрагивается.
- Отменено физическое удаление строк (из-за риска потери блоков сотрудников в merged-шаблоне), используется безопасное скрытие разрывов.
- Версия обновлена до `1.4.20`.

## 1.4.19 (2026-04-04)
- Исправлено заполнение в скрытые колонки Т-13:
  - определение колонок дней (`1..31`) теперь игнорирует `hidden`-колонки шаблона;
  - запись явок/часов выполняется только в видимые колонки дневной сетки.
- На листе `Т-13 общий` удаляется промежуточный блок строк `106..133` (служебная шапка/подвал между страницами),
  чтобы список сотрудников шел непрерывно.
- Версия обновлена до `1.4.19`.

## 1.4.18 (2026-04-04)
- Стабилизирована верстка Т-13:
  - отключены операции, которые меняли геометрию шаблона (растягивание/пересборка merge-структуры в табличной части);
  - заполнение возвращено в режим «только запись значений в существующие ячейки шаблона».
- Для `Т-13 общий` отключено принудительное наращивание непрерывной таблицы, чтобы исключить сдвиг толстых границ и “разъезд” колонок.
- Проверка после фикса: количество merged-диапазонов на листе `Т-13 общий` совпадает с исходным шаблоном (`5258`), что подтверждает сохранение структуры бланка.

## 1.4.17 (2026-04-04)
- Улучшена читаемость `Матрица графика`:
  - колонки `* Явка` выравниваются по правому краю;
  - колонки `* Часы` и `Итого часов` выравниваются по левому краю.
- Улучшено отображение значений в Т-13 для узких ячеек дней:
  - для дневной сетки включен `shrink-to-fit` и центровка;
  - минимальная ширина колонок дней увеличена до читаемой (`>= 2.8`).
- Дополнительно закреплено безопасное применение выравнивания в merge-ячейках Т-13 через запись в anchor-ячейку.
- Версия обновлена до `1.4.17`.

## 1.4.16 (2026-04-04)
- Синхронизирован `Скачать график` с Т-13:
  - лист `Матрица графика` теперь строится из той же логики, что и Т-13 (те же выбранные дни/часы по сотруднику);
  - устранено расхождение данных между `schedule_*.xlsx` и `timesheet_t13_*.xlsx`.
- Исправлен лист `Т-13 общий`:
  - включен непрерывный общий список сотрудников (без повторных шапок страниц);
  - очищаются значения подразделений в верхней части листа, чтобы общий лист не содержал привязки к одной точке.
- Для блоков сотрудников снова включена нормализация merge в сетке дней, чтобы не пропадали часы в “узких” объединенных ячейках.
- Версия приложения обновлена до `1.4.16`.

## 1.4.15 (2026-04-04)
- Откат шаблона Т-13 на предыдущий рабочий `templates/t13_template.xlsx` (без “нового” проблемного шаблона).
- Обновлен файл по кнопке `Скачать график`:
  - основной лист `Матрица графика` теперь в формате:
    - строки: сотрудники;
    - по каждому дню две колонки: `Явка` и `Часы`;
    - справа итоги по сотруднику: `Итого смен`, `Итого часов`;
    - внизу добавлена строка `ИТОГО ПО ДНЯМ` (число работающих и суммарные часы по каждому дню).
  - добавлены служебные листы `График (сырье)` и `Матрица (тех.)` для проверки данных.
- Повышена версия приложения до `1.4.15` (endpoint `/version`).

## 1.4.14 (2026-04-03)
- Исправлен разбор расчетных листков для Т-13:
  - приоритет отдан фактическим строкам оплаты/отработки (`Оплата по окладу`, `Рабочие`), а не норме времени;
  - строки `Оклад (тариф)` и `норма` больше не подменяют фактические `дни/часы`.
- Добавлен персонифицированный приоритет половины месяца:
  - для каждого сотрудника определяется `first/second/neutral` по суммам выплат за половины месяца;
  - при формировании дней в preview и Т-13 распределение сдвигается в нужную половину месяца.
- В экспорт Т-13 добавлен второй лист:
  - `Т-13 общий` — сплошной список сотрудников по алфавиту (без группировки по подразделениям).

## 1.4.7 (2026-04-02)
- Упрощен сценарий загрузки: файл `Табель` больше не обязателен, основной источник — расчетные листки.
- В `POST /upload` оставлен обязательным только `payroll_file` (плюс опциональный `employees_file`).
- Экспорт Т-13 переведен на встроенный формат (без шаблонного файла табеля).
- Календарь для генерации при отсутствии табеля берется как полный месяц `1..31` с базовой эвристикой выходных.

## 1.4.6 (2026-04-02)

- Логика покрытия групп по ресторанам закреплена:

  - если в группе нет доступных сотрудников, это отображается как дефицит.
- В предпросмотре скрыта колонка `Время`.

## 1.4.5 (2026-04-02)

## 1.4.13 (2026-04-02)

## 1.4.9 (2026-04-02)

## 1.4.8 (2026-04-02)

- Источник праздничного календаря: библиотека `holidays` (`RU`, `observed=True`) с fallback на фиксированные федеральные даты.
- Добавлены/зафиксированы дефолтные группы ролей:
  - `бар-менеджер` -> `Бар`;
  - `директор` -> `Зал/Касса`;
  - `управляющий по производству и учету` -> `Кухня`;
  - `подсобный рабочий` -> `Кухня`.
