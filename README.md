# Генератор графика сотрудников (FastAPI + Frontend)

Веб-приложение для генерации рабочего графика на основе 2 Excel-файлов:

1. `Расчетные листки` — лимиты по часам и дням на сотрудника.
2. `Табель` — структура дней месяца.

Подразделение (ресторан) и должность берутся напрямую из расчетных листков.

Приложение генерирует график с учетом ограничений:

- максимум `13` часов в день на сотрудника;
- максимум `6` рабочих дней подряд (для более равномерного графика);
- суммарные часы сотрудника `<=` лимита из расчетного листка;
- суммарные дни сотрудника `<=` лимита из расчетного листка;
- смены:
  - `бизнес-ланч` `12:00-16:00`;
  - `вечерняя посадка` `18:00-22:00`;
- в выходные покрытие усиливается (больше слотов), в будни — меньше;
- используется жадный алгоритм с балансировкой нагрузки.
- взаимозаменяемые роли:
  - `администратор`, `администратор-кассир`, `кассир`, `менеджер`, `управляющий` -> `админ/касса/управление`;
  - `повар`, `шеф`, `су-шеф` -> `кухня (повар/шеф/су-шеф)`.
  - `бармен`, `бар-менеджер` -> `бар (бармен/бар-менеджер)`;
  - `официант`, `хостес`, `раннер` -> `зал (официант/хостес/раннер)`.
- необязательные роли (дефицит по ним не считается ошибкой покрытия):
  - `подсобный рабочий`, `бухгалтер`, `директор`, `маркетолог`, `аниматор`, `гардеробщик`;
  - `аниматор` и `гардеробщик` смещены в выходные;
  - `бухгалтер`, `подсобный рабочий`, `маркетолог` смещены в будни.
- для кухни разрешена межресторанная взаимозаменяемость (если в своем ресторане нет доступных людей).
- при дефиците система не падает: создает график с пометками `дефицит` и предупреждениями.

## Структура проекта

- `app/main.py` — FastAPI API и маршруты `/upload`, `/generate`, `/download`, `/download_t13`, `/version`, `/download_log`.
- `app/excel_parsers.py` — парсинг Excel через `pandas`.
- `app/scheduler.py` — генератор расписания и экспорт в Excel.
- `app/logging_utils.py` — файловое логирование и хранение tracebacks ошибок.
- `static/` — простой русский frontend (HTML/CSS/JS).
- `outputs/` — сохраненные результаты генерации.
- `logs/` — рабочие логи (`app.log`) и отдельные файлы ошибок.
- `VERSION` и `versions.md` — версия приложения и история изменений.

## Запуск локально

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

## API

### `POST /upload`
Загрузка расчетных листков (`multipart/form-data`):

- `payroll_file`
- `employees_file` (необязательный, если хотите переопределять подразделение/роль из отдельного файла)

Ответ: сводка распознанных данных и предупреждения.

### `POST /generate`
Генерация графика на основе загруженных файлов.

Ответ: статистика, предупреждения и первая страница preview назначений.

### `GET /preview?offset=&limit=`
Постраничная подгрузка предпросмотра сгенерированных назначений:
- `offset` — с какого индекса читать;
- `limit` — сколько строк вернуть.

### `GET /download`
Скачивание последнего сгенерированного файла `.xlsx`.

### `GET /download_t13`
Скачивание отдельного файла табеля в формате `Унифицированная форма № Т-13`.

### `GET /version`
Возвращает текущую версию приложения.

### `GET /download_log`
Скачивание `logs/app.log`.

### `GET /download_error_log/{error_id}`
Скачивание отдельного файла ошибки `logs/errors/{error_id}.log`.

## Формат результата Excel

Файл содержит 3 листа:

- `График` — список назначений по дням/сменам;
- `Сводка` — итоги по каждому сотруднику (план/лимиты);
- `Матрица` — табличное представление по дням (`БЛ`, `В` и др.).

Отдельно доступен второй файл:

- `Т-13` — табель учета рабочего времени по сотрудникам (коды явок и часы по дням).
  - формируется на основе встроенного шаблона проекта `templates/t13_template.xlsx` (сохраняется внешний вид формы);
  - целевые `дни` и `часы` сотрудника берутся из расчетного листка (`max_days` / `max_hours`);
  - фактические назначенные смены помечаются `Я`;
  - при межресторанной подмене в ячейку `Я` добавляется примечание (комментарий Excel);
  - если день попал в табель только из расчетного листка без назначенной смены генератора — ставится `ПЛ`.

## Примечания

- Для расчетных листков поддерживается блочный формат (по сотруднику), из которого извлекаются `часы`, `дни`, `подразделение` и `должность`.
- Поиск колонок в Excel гибкий: поддерживаются русские и английские названия (`сотрудник`, `фио`, `hours`, `days`, `ресторан`, `роль` и т.д.).
- Если ресурсов по группе `ресторан + роль` не хватает для покрытия обязательных смен, генерация не падает: дефицитные смены фиксируются в результате с пометкой `дефицит`.
- Для необязательных ролей (например, бухгалтер/маркетолог/подсобный рабочий, аниматор/гардеробщик) дефицит не считается обязательным нарушением покрытия.
- В веб-интерфейсе доступен ручной редактор групп взаимозаменяемости должностей (`Кухня`, `Зал`, `Касса`, `Бар`, `Обслуживание`), и выбранные группы учитываются при генерации.

## Версионирование

- Актуальная версия хранится в файле `VERSION`.
- История изменений ведется в `versions.md`.
- Версия доступна в API: `GET /version`.

## Логирование и диагностика

- Общий лог: `logs/app.log`.
- Для внутренних ошибок автоматически создается отдельный файл в `logs/errors/ERR_*.log`.
- В тексте API-ошибки возвращается `Код ошибки` и путь к логу, чтобы быстро найти traceback.

## Запуск онлайн (Render/Railway)

Можно подключить GitHub-репозиторий и запускать как Web Service:

1. Build command: `pip install -r requirements.txt`
2. Start command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
3. Переменные окружения (опционально): `HOST`, `PORT`

Скрипт `run.sh` уже поддерживает `PORT`, поэтому можно запускать и через него на облаке.
